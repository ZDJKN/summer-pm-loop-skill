# summer-pm-loop-skill · PM loop skill

> 一个**像老练 PM 一样**陪你跑项目的 Claude skill：接 brief 帮你拆、派活帮你传达、提审帮你体检、客户卡住帮你救火、项目跑偏帮你盘点。
> **它的「啊哈」**：把一个项目丢给它 → 自动 onboard（抓人/事/流程）→ 用一套 PM 工作标准给项目做体检 → 直接告诉你「哪健康、哪要爆、下一步干嘛」。

作者：**ZDJKN · Summer · Mr Wu**

---

## 1. 它能干嘛

说出场景就触发，不用记命令：

| 你说 | 它做 |
|---|---|
| 「帮我拆这个 brief / 这单接不接」 | brief 必问4维 + 解读4步 + 一句话复述 |
| 「算一下人力毛利 / 报多少」 | 团队滚动口径算人力 + 毛利测算（<30% 不接预警） |
| 「这条能发吗 / 提审前帮我审」 | 内容7维校验 + 客户 challenge |
| 「客户又改需求了」 | 暂停 → 评估三影响 → 给选择题 → 留痕 |
| 「客户不满意 / 救火 / 客户不回」 | 救火12步 · 等待坑给选择题 · 升级留痕 |
| 「写周报 / 报风险 / 怎么跟客户说」 | 结论先行 · PREP · 情境-影响-方案 话术 |
| 「盘一下这个项目 / 有没有风险」 | 七维健康灯 + 火苗信号扫描 |
| 「复盘一下」 | 心流复盘两问 → 资产化 |

## 2. 心智模型

```
知识库（真理来源）        skill（执行动作）        loop（三阶段）
6 模块 PM 标准      →   读知识库 → 陪你跑 checklist   →   ①抓取 ②盘点 ③滚动
```

- **知识库是真理来源，skill 只把知识变成动作。** 改标准 = 改 `knowledge/` 里的 md，skill 自动跟随，**不用动 skill 代码**。
- **loop 三阶段**：① 抓取（像 PM 抓 人/事/流程/背景 → 建项目文件夹）→ ② 盘点（拿知识库体检 → 健康灯+卡点+下一步+预警）→ ③ 滚动（重抓→更新→出 diff）。
- 每个项目一个**动态知识包** `projects/<项目>/`（00总览/01背景/02人/03事/04流程/05卡点/06留痕），loop 不断回写它。

## 3. 安装

1. 把整个文件夹放进 `~/.claude/skills/`。
2. 新开一个 Claude Code 会话，确认技能列表里能看到它。
3. 直接说场景（见上表）即可触发。

> 路径按你的 agent 来：Claude Code 放 `~/.claude/skills/`，Codex 放 `~/.codex/skills/`。
> 跑起来最少只需「能读写文件的 agent + 本包」，核心动作零外部依赖。想要自动抓群聊/表格再按 [`INSTALL.md`](INSTALL.md) 装对应连接器。

### 首次使用 · 环境自检（建议先跑）

第一次用 / 换人接手时，对它说「**首次使用 / 怎么准备**」，它会按 [`workflows/preflight.md`](workflows/preflight.md) **自检环境 → 探测已装应用（Obsidian / lark-cli / cron…）→ 给你一份「就绪报告」+ 缺啥补啥的命令**。其中可选工具（如 Obsidian）会**先问你再帮你装**（`brew install --cask obsidian` 之类），不静默安装。核心知识库零依赖，不装这些也能用。

## 4. 改成你自己的（配置点）

| 配置点 | 在哪 | 改成 |
|---|---|---|
| 知识库 | `knowledge/`（包内，相对路径） | 一般不用改；想改 SOP 直接改这里的 md |
| 项目档案位置 | `projects/`（或你自己的 Obsidian / 飞书多维表格） | 你的项目台账位置 |
| 项目档案模板 | `projects/_TEMPLATE/`（00-06 空表） | 复制一份按真实项目改名填充 |

> `knowledge/` 是**通用 PM 标准**，可直接复用；你的**真实项目数据**（客户名/群ID/毛利数）只放 `projects/<项目>/`，已被 `.gitignore` 挡住，**不会进公开仓库**。

## 5. 现状 & 路线（诚实版）

**已经能用：**
- 知识库 6 模块（目标管理 / 运行管理 / 毛利经营 / 风险与救火 / 复盘沉淀 / 对客沟通），原子化、带双链+标签。
- 节点能力按路由读知识库跑（见 §1）。
- **抓取 & 盘点工作流已写进 skill**（`workflows/capture.md` + `workflows/preflight.md`，路由表已挂）：被触发时按决策树抓「人/事/流程」建档、再用 PM 标准体检出健康灯。

**还没做（路线）：**
- ⬜ **全自动 loop**：现在是「你喊它才跑」的半自动；**定时、无人值守的自动抓取 + 日报（阶段③滚动）还没做** → 计划基于 [cn-messaging-context](https://github.com/Iii3pl/cn-messaging-context)（群聊抓取/摘要/定时后端）+ 定时器实现。
  - **为什么先不做全自动**：① 自动化是放大器——先得让被放大的「判断质量」稳（知识库夯实 + 实战验证），否则自动发错日报、误判健康灯，比不发更糟；② 抓取/定时不自己造轮子，等专用后端验证稳了再接，更省也更准。路线是**先把判断做扎实 → 再上自动化**，不是反过来。
- ⬜ **自学习**：把外部职场内容蒸馏进知识库（带闸门，只收可执行动作）。

---

## 6. 维护方式

- **改 SOP** → 改 `knowledge/` 的 md，skill 自动跟随。
- **实战反哺**：真实项目跑完，把新坑 / 新话术沉淀回知识库——这套就是这么长出来的。
- **外部信息摄取**要过**蒸馏闸门**：只收「可执行的动作 / checklist / 公式」，鸡汤 / 情绪价值丢弃，否则会稀释。

