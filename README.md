# xiaoti-pm · PM loop skill（交接说明）

> 一个**像老练 PM 一样**陪你跑项目的 Claude skill：接到 brief 帮你拆、要派活帮你传达、提审帮你体检、客户卡住帮你救火、项目跑偏帮你盘点。
> **它的「啊哈」**：把一个项目丢给它 → 它自动 onboard（抓人/事/流程）→ 拿一套 PM 工作标准给项目做体检 → 直接告诉你「哪健康、哪要爆、下一步干嘛」。
> 本文档面向**接手/使用这个 skill 的人**。作者：Summer。

---

## 1. 它能干嘛（节点能力）

说出场景就触发，不用记命令：

| 你说 | 它做 |
|---|---|
| 「帮我拆这个 brief / 这单接不接」 | brief 必问4维 + 解读4步 + 一句话复述 + 信息清晰度盘点 |
| 「算一下人力毛利 / 报多少」 | 团队滚动口径算人力 + 毛利三步（<30% 不接预警）|
| 「这条能发吗 / 提审前帮我审」 | 内容7维校验 + 客户 challenge q1-q6 |
| 「客户又改需求了」 | 暂停→评估三影响→给选择题→四件套留痕 |
| 「客户不满意 / 救火」 | 满意度救火12步 |
| 「客户不回 / 催反馈」 | 等待坑：给选择题 + 默认动作 + 升级留痕 |
| 「盘一下这个项目 / 有没有风险」 | 七维健康灯 + 火苗信号扫描 |
| 「复盘一下」 | 心流复盘两问 → 资产化 |

> **实战印证**：2026-06-24 用它跑「某短视频代运营项目」项目一整天——客户连续失约救火、给选择题、双线进度同步、新人对接话术，全程在线。

## 2. 心智模型（为什么它靠谱）

```
知识库（真理来源）          skill（执行动作）           loop（自动盯盘）
26.01~26.05 五模块 SOP  →  读知识库 → 陪你跑checklist  →  ①抓取②盘点③滚动
```

- **知识库是真理来源，skill 只把知识变成动作。** 改标准 = 改知识库的 `26.0x`，skill 自动跟随，**不用动 skill 代码**。
- **loop 三阶段**（v1 在建）：① 立项抓取（像 PM 抓 人/事/流程/背景 → 建项目文件夹）→ ② SOP 盘点（拿知识库体检 → 七维健康灯+卡点+下一步+预警）→ ③ 滚动重抓、更新、出 diff。
- 每个项目一个**动态知识包** `_项目档案/<项目名>/`（00总览/01背景/02人/03事/04流程/05卡点/06留痕），loop 不断回写它。

## 3. 安装（拿到包之后）

1. 把整个 `xiaoti-pm/` 文件夹放进 `~/.claude/skills/`。
2. 新开一个 Claude Code 会话，确认技能列表里能看到 `xiaoti-pm`。
3. 直接说场景（见上表）即可触发。

## 4. 改成你自己的（⭐ 分享关键 · 配置点）

这个包默认带着「小题」的设定，接手后改这几处就成你的：

| 配置点 | 在哪 | 改成 |
|---|---|---|
| **知识库路径** | `SKILL.md`「知识库」一节 | v1 起知识库**打包在 `knowledge/` 内**，用相对路径，无需改（见 §6）|
| **项目档案位置** | `SKILL.md`「留痕原则」| 你自己的项目台账（Obsidian 文件夹 / 飞书多维表格）|
| **团队 & 客户信息** | `_项目档案/<你的项目>/` | 删掉示例「某短视频代运营项目」，按你的真实项目新建 |
| **团队成员名/角色** | 各项目档案 02 人.md | 你的 RACI |

> 知识库里的 SOP（26.01~26.05）是**通用 PM 标准**，不含小题机密，可直接复用；机密只在 `_项目档案/` 里，分享时**剔除**。

## 7. 维护方式

- **改 SOP** → 改知识库 `26.0x`，skill 自动跟随。
- **实战反哺**：每次真实项目跑完，把新坑/新话术沉淀回知识库（这套就是这么长出来的）。
- **外部信息摄取**（如抖音职场内容）要过**蒸馏闸门**：只有「可执行的动作/checklist/公式」才进知识库，鸡汤/情绪价值丢弃，否则会稀释。
