# 安装与依赖 · xiaoti-pm PM loop skill

> 本文档面向**接手这个 skill 的人或 AI agent**。读完你就知道：跑起来最少需要什么、想要哪个能力补哪个工具、缺了怎么优雅降级。
> 设计原则：**核心零依赖，增强按需装。** 别被长清单吓到——Tier 0 就能用。

---

## 0. 30 秒最小可运行

```
1. 把整个 xiaoti-pm/ 文件夹放进  ~/.claude/skills/
2. 用一个有「读写本地文件」能力的 agent 环境（Claude Code / Claude Agent SDK / 任何支持 Agent Skills 的 harness）
3. 说出场景（"帮我拆这个 brief" / "客户不回怎么办" / "盘一下这个项目"）即触发
```
核心动作只读 `knowledge/` 里的 markdown + 跟你对话，**不需要任何外部工具或联网**。

## 1. 能力 ↔ 依赖 矩阵（缺了怎么退化）

| 能力 | 必需？ | 靠什么 | 缺了的降级方案 |
|---|---|---|---|
| 节点动作（拆brief/7维校验/毛利测算/救火话术/复盘） | ✅ Tier0 | 文件读写 + 对话 | —（本就零依赖） |
| loop 阶段①**抓取**项目信息（群成员/消息/表格） | 🟡 Tier1 | 平台连接器（见 §3） | 你手动把信息粘给它，抓取变人工 |
| loop 阶段②**盘点**（七维健康灯/卡点/预警） | ✅ Tier0 | 读知识库 + 项目档案 | — |
| loop 阶段③**滚动/定时自动跑** | 🟡 Tier1 | 定时器（/loop 或 cron） | 你手动每天喊一次"盘一下" |
| v2 自学习（抓职场内容/转写/蒸馏/人格提炼） | ⬜ Tier2 | 见 §4 | 不影响 v1，整块可不装 |

## 2. Tier 0 · 必需

| 项 | 说明 |
|---|---|
| **Agent 运行环境** | 任何支持「Agent Skills + 文件 Read/Write/Edit」的 harness。参考实现：Claude Code。 |
| **本 skill 包** | `SKILL.md`（节点路由+流程）、`knowledge/`（打包的 5 模块 SOP）、`templates/`、本文件。 |

> ⚠️ 打包要求：`knowledge/` 必须随包携带，`SKILL.md` 用**相对路径**引用它（不要写绝对路径），否则换台机器读不到知识库。

---

## 3. Tier 1 · 推荐（让「抓取」自动化 · 按你项目所在平台选）

阶段①要把项目的「人/事/流程」抓进档案。装与你**项目所在平台**匹配的连接器即可，**多选或不选都行**：

| 你的项目在 | 装什么 | 提供 |
|---|---|---|
| **飞书 / Lark** | `lark-cli`（lark-im / lark-base / lark-doc / lark-wiki 等 skill） | 抓群成员/消息、读多维表格、读文档 |
| **钉钉** | `dws` / dingtalk-scraper | 群聊/表格/日志抓取 |
| **Slack / Notion / Asana / Atlassian(Jira/Confluence)** | 对应官方 MCP connector | 频道消息 / 页面 / 任务 |
| **通用网页 / 社媒（已登录 Chrome）** | `opencli` + Chrome 扩展，或 browser MCP | 任意站点抓取 |
| **定时跑 loop** | Claude Code `/loop`，或系统 cron / scheduled-tasks | 周期触发健康盘点 |

> 没有任何连接器也能用——把群消息/表格截图或文本**粘给它**，阶段①变手动，其余照常。

## 4. Tier 2 · 可选（v2 自学习 / 体验增强 · v1 可全部跳过）

| 想要 | 装什么 |
|---|---|
| 自动抓职场内容来学习 | `opencli`（抖音/文章）+ 蒸馏闸门（只收"可执行动作"，鸡汤丢弃） |
| 音视频转文字 | `lark-minutes` 或 whisper |
| 从对话提炼对话者人格 | `huashu-nuwa` skill |
| 双链 + 标签可视化信息 map | Obsidian（**非必需**，知识库是纯 markdown，任何编辑器可读） |
| 灵感/碎片沉淀 | flomo |

## 5. 给「接手 agent」的自检与自处理流程

接手的 agent 按此自我配置（可直接执行）：

```
STEP 1 · 确认核心
  - 检查 ./knowledge/ 是否存在 5 个模块 md；不存在 → 报"知识库未打包，无法运行"。
  - 检查能否 Read/Write 文件；不能 → 报"缺文件能力"。

STEP 2 · 探测可选连接器（决定抓取能力）
  - 项目在飞书？ 试 `lark-cli --version`；有 → 阶段①可自动抓；无 → 走手动粘贴。
  - 项目在钉钉/Slack/Notion？ 检查对应 skill/MCP 是否在可用列表里。
  - 都没有 → 不报错，告知用户"抓取走手动，把群消息/表格粘给我即可"。

STEP 3 · 定位知识库与项目档案（见 §6 配置点）

STEP 4 · 自检通过后，按 SKILL.md 的节点路由表工作。
```

**降级原则**：任何 Tier1/Tier2 工具缺失都**不得报错退出**，一律降级为手动 + 明确告知用户缺哪个、影响什么。

## 6. 配置点（改成你自己的）

| 配置 | 默认 | 改成 |
|---|---|---|
| 知识库路径 | `./knowledge/`（包内，相对路径） | 一般不用改 |
| 项目档案位置 | `./projects/`（或你自己的 Obsidian 文件夹 / 飞书多维表格） | 你的项目台账位置 |
| 项目档案模板 | `./projects/_TEMPLATE/`（00-06 空表） | 复制一份按真实项目改名填充 |

> 🔒 **隐私**：真实项目档案（客户名/群ID/毛利数）**只放本地，永不进公开仓库**——`.gitignore` 已挡 `projects/`，公开仓库只保留 `_TEMPLATE/`。

---

## 7. 验证安装成功

对 agent 说任意一句：
- 「帮我拆这个 brief：客户想做一支品牌视频」→ 应触发 brief 必问4维 + 一句话复述。
- 「客户不回消息怎么推进」→ 应给"等待坑：选择题 + 默认动作 + 升级留痕"。

能正确路由到知识库动作，即安装成功。

---

*依赖哲学：这套东西的真理在 `knowledge/` 的 markdown 里，工具只是把信息抓进来、把动作跑出去的手脚。手脚缺了能用嘴（手动），脑子（知识库）不能缺。*
