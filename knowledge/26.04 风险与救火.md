# 26.04 风险与救火 🔥

> 灭火第一要务是「**不让火势扩大**」——先稳局面，再解决问题。
> 火星也要看到，不然下次看到就是起火的时候。
> 上级 → [[26 项目管理PM]] ｜ 信源：《PM工作标准》01·三 场景延展
> 🟢 Phase 1 已原子化｜格式：场景 / ❌误区 / ✅标准动作 / 🚨预警 / 📌留痕

---

## 总纲 · 救火两种结局

| 结局 | 特征 |
|---|---|
| 🔥 火势更大 | 多头响应、职责不清、信息断层、协同混乱——**缺乏统一指挥** |
| ✅ 成功灭火 | 一人统筹、及时同步、预期管理、闭环确认 |

> 稳住局面的核心不是火速给完美方案，而是**释放「我们在处理」的信号** + 明确责任人 + 同步解决思路，不让客户在盲目等待中失去耐心。

---

## E1. 成本三隐形坑（事中常态防御）

- **场景**：日常执行，防患于未然。除看得见的人力，**返工/沟通/等待**是三大隐形杀手。
- **✅标准动作**：

| 坑 | 典型表现 | 标准动作 |
|---|---|---|
| 返工坑 | 同一东西改2版+ | 接单即确认·确认即留痕·嘴巴多问。开工前**灵魂五问**：给谁看？什么场景？想要什么效果？人力消耗多少？有无明确参考？ |
| 沟通坑 | 一件事说几遍没对齐 | 群里来回>3轮无结论→立刻拉会；会议结束前当场复述结论；会后立刻发纪要（共识+待办+标准+责任人+时间点） |
| 等待坑 | 干等反馈被拖死 | 任务发出同步问清"最晚什么时候给反馈"；节点前1天提醒；超时立刻催并告知延期影响 |

- **兜底法则**：标准动作能避一大半坑，剩下20%靠命——别硬扛，立刻**向上反馈求助 + 疯狂留痕**。流程是护甲，至少不背锅。
- **📌留痕**：确认记录 + 会议纪要 + 催办记录。

## E2. 看到火苗（信号→处理）

- **场景**：客户不满往往不直说，信号藏在话里。听懂叫掐灭火苗，听不懂叫事故重演。
- **✅标准动作（信号识别表）**：

| 信号 | 潜台词 | 处理 |
|---|---|---|
| "不太对" | 方向偏了 | 立即停，问清"不对"的具体点，内部重新拉齐再动 |
| 同一问题反复提 | 你们没重视 | 拉会/专项复盘：解决思路 + 到位时间 + 预期效果 |
| 开始追问细节 | 信任预警（在查作业） | 带方案回复：概念+重点+思考逻辑+画面逻辑+数据支撑 |
| 提"领导在看/关注" | 重视升级 | 升级处理、加资源人手、拉总监，用行动表态 |
| 情绪急躁 | 等太久了 | 先安抚再给确定性：谁+何时+什么标准+可能风险+应对预案 |

- **📌留痕**：火苗处理记录。

## E3. 满意度救火 SOP（12 步法）

- **场景**：客户情绪爆发、集中反馈不满、领导否定方案、信任危机。
- **核心原则**：先处理情绪再处理问题；先拉齐需求再动手执行。

**第一阶段·接住情绪&拆解（1-4）**
1. **情绪承接**：负面反馈后10分钟内必有人回应；不解释不反驳不说"但是"；明确告知"会给方案+具体时间点"；务必复述确认"我理解的对不对"。
2. **PM第一轮**：客户反馈原封不动整理；自问 不满什么 / 核心诉求是什么 / 诉求合理吗能满足吗。
3. **PM第二轮**：把"情绪话"翻译成"需求话"（"太丑"=哪帧？什么颜色？哪段节奏？跟预期差在哪）。**没转译就传达=做好50%失败的准备。**
4. **PM第三轮**：快速拉2-3个参考（不同维度，不是同类找多张）。没经验不慌：一问AI，二呼叫SOS——寻求协助不丢人。

**第二阶段·对齐方向&下达（5-6）**
5. **下达需求四要素齐**：① 谁（角色+覆盖范围）② 数量（找几个参考/改几版/几个方向）③ 标准（附参考图，说明色彩/氛围/风格/光影/运镜/构图/质感）④ 时间（精确到小时）。**没参考=没标准=反复改。**
6. **内部对齐**：执行人复述"我理解要做xxx，参考是xxx，对吗"，拿到确认再动手；复杂需求拉5分钟站会。

**第三阶段·确认&落地（7-9）**
7. **给客户确认参考方向**：发方向"确认后开始执行"；不用等完美参考包，一两张图可短平快试探；**带着方向去确认，别空手问"你觉得怎么做"**。
8. **执行前二次确认**：设计/剪辑/对接人多方确认 效果是什么/参考是哪个/交付标准。
9. **执行跟进**：每2小时同步一次；发现问题立刻停下来升级+同步全员；调方向必同步客户三件事（新方向+完成时间+交付时间）。**永远不要让客户在无止境等待中失去耐心。**

**第四阶段·闭环&复盘（10-12）**
10. **交付带"对比包"**：旧版+新版+参考图+修改说明（改了哪/为什么/优化逻辑/效果）。
11. **闭环反馈**：拿到明确确认"可以/过/好多了"；若"再调调"回到Step4重走；截图存客户原话。
12. **随时心流复盘**：解决完立刻复盘，趁记忆还热（满意度升→可复用沉淀SOP；降→哪做错了）。结论必含一条"下次遇到类似第一步做什么"。→ 见 [[26.05 复盘沉淀]]

- **📌留痕**：12步全程（尤其Step1响应、Step5四要素、Step11客户确认截图）。

## E4. 客户情绪上头·处理公式

- **场景**：客户激动、不知道/不敢回复。基本原则：只要不硬犟、不回避，一般不会有大问题。
- **✅标准动作**：**放低姿态认错 + 具体解决思路 + 表示重视尊重**。
  - 流程：认错/哪里错 → 表示认同理解 → 转移视线·拆解需求和动作 → 表达尊重重视 →（情况允许）增加重视动作（协调总监/重新调配人力/加人手）。
  - 要点：时间承诺预留缓冲（避免过度承诺）；姿态到位但不卑躬屈膝聚焦解决；动作落地有具体修改+时间节点。
- 💡 认错要干脆，动作要实在，像谈恋爱一样谈客户。
- **📌留痕**：沟通记录。

## E5. 节假日值班

- **场景**：值班、交接。不要"我以为你理解了"。
- **✅标准动作**：
  - **值班前交接**：项目状态（阶段/待办/关键节点）/ 客户背景（对接人/重视度/是否领导关注）/ 已知风险 / 交付标准（附参考案例）/ 应急联络（书面名单）。
  - **响应原则**：客户新需求→10分钟内"已收到正在协调"+同步负责人不立即承诺；反馈问题→10分钟内回+15分钟内初步判断或升级；情绪激动→先承接+同步应急人；无法判断→不擅自处理立即升级。
- **🚨值班红线**：收到消息延迟回复超30分钟 / 擅自承诺交付时间 / 问题私下消化不同步 / 假装处理实际没动作。
- **📌留痕**：值班交接文档 + 应急联络名单。
