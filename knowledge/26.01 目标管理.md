# 26.01 目标管理 🎯

> 运营目标不是「把项目做完」，而是「做到什么程度才算成功」。
> **目标支配一切运营行为，不拉齐 = 方向盘松了**，做好多次返工的准备。
> 上级 → [[26 项目管理PM]] ｜ 信源：《PM工作标准》02 目标管理（+ 飞书 PM工作标准类文档待并）
> 🟢 Phase 1 已原子化｜格式：场景 / ❌误区 / ✅标准动作 / 🚨预警 / 📌留痕

---

## 总纲

- **目标公式**：`[什么时间周期] + [通过什么核心策略] + 实现 [什么核心指标] 的 [提升/变化]`
  例：本月 + 通过3条短视频 + 实现涨粉5000。
- **目标四侧**（每个项目都要四侧齐看）：

| 侧 | 定义 | 内容示例 |
|---|---|---|
| 交付侧 | 作业成果 | 做什么/怎么做/做多少/有无规范 |
| 效果侧 | 数据成果 | 传播/粉丝/舆论/UGC亮点/爆款 |
| 体验侧 | 服务成果 | 满意度/需求理解/沟通顺畅/专业支持/汇报质量 |
| 商业侧 | 转化成果 | 引流/转化/线索/业务价值/内部认可/声量 |

- **总目标 vs 流动性阶段目标**：总目标从 brief 延展/每季度定，全员书面同步留痕；阶段目标随实际情况（满意度/需求调整/成本波动/推进困难）灵活拆解，不固定按周/月。

> 救火案例（某剪辑工具客户）：因误判客户「内容质量」核心痛点引发满意度危机 → 立即调阶段目标：「2周内」通过「强化出品+寻求资源协助」实现「画面质量/客户满意度/专业度感知」，维稳后再优化数据。

# A. 目标是什么 · 怎样算好

## A1. 目标只有统一·作业才有意义

- **场景**：项目初期、每个阶段开始。
- **❌误区**：以为目标是「默认的」，只统计数据即可。**数据统计只代表进度完成度，不代表目标完成度**（进度=做到哪一步，目标=方向对不对）。
- **✅标准动作**：总目标全员收到+统一认知+书面留痕；阶段目标随项目情况及时调整。
- **🚨预警**：各职能角色各有各的理解、力不往一处使。
- **📌留痕**：目标书面同步（群公告/文档/纪要）。

## A2. 好的目标管理四维

| 维度 | 误区 → 正解 | 动作 |
|---|---|---|
| **全员可视化** | 以为目标是默认的 → 目标完成度≠进度完成度 | 目标出现在 工作汇报/PM每日todo/项目群公告/SOP/脚本文档/会议纪要 |
| **服务项目健康流转** | 好项目=没问题 → **没有100%健康的项目**，目标把控=随时检查健康状态 | 重视并拆解客户反馈、按问题大小调阶段目标、用目标重设聚拢注意力 |
| **促进团队稳定发展** | 保证完成度第一 → 在达标与团队健康间找平衡 | 目标过高→动作变形；过低→能力停滞；**适中=有压力但可控**。好把控 = 缓解焦虑 + 刺激活跃度 |
| **符合业务目标** | — | 内外都符合→必做；不符内部符客户→衡量后做；符内部不符客户→推荐做 |

- **🚨预警**：只看数据、只催进度；阶段目标达成后不复盘不给反馈。
- **📌留痕**：可视化看板/文档/群公告。

# B. 三向同步（核心差别）

> **向上对齐资源、向下对齐执行、对客对齐预期。** 三方核心都是——让对方感到「这事我心里有数、手里有方案、过程可控」。

## B1. 向上同步（老板/+1 关注什么）

- **场景**：立项、目标调整、要资源时向上汇报。
- **核心逻辑**：老板每个问题背后都在评估「投入产出比」和「风险边界」。
- **✅标准动作（7维度，结论先行）**：① 是什么（一句话说清目标）② 怎么做（拆关键动作展示逻辑链）③ 利润空间（带前后%数据）④ 效果评估（量化指标+概率判断）⑤ 全盘影响（资源占用/机会成本）⑥ 长线价值（复购/口碑/案例价值）⑦ 费用挖掘（增收机会/预算弹性）
  - 目标调整时向上传达需覆盖：调整背景 / 影响评估 / 执行方案 / 长期判断。
- **🚨预警**：只报"做完了"不带判断；汇报没有数据对比和方案。
- **📌留痕**：目标确认纪要（与+1/老板书面确认）。
- ⚠️ 切忌死抄模板，具体问题具体分析。

## B2. 向下同步（帮团队高效执行）

- **场景**：项目启动会、派活、目标下达。
- **核心逻辑**：向下同步不是「通知」，是「对齐认知 + 标定执行边界」。
- **✅标准动作（7要点）**：① 目标转译（抽象目标→可执行动作）② 背景同步（为什么做+认可案例+绝对no案例+历史问题）③ 边界清晰（权限/资源上限/红线/重点）④ 标准对齐（参考案例+验收标准+时间节点）⑤ 职责明确（谁负责怎么配合）⑥ 预案前置（异常处理提前告知）⑦ 反馈机制（什么情况上报/什么情况自行决策）
  - PM心里必须清楚：拆解到人（谁负责/配合/兜底）、提供工具支撑（模板/checklist/参考）、关键节点检查（设同步节点非全程盯）、容错空间（哪些可试错/哪些必请示）。
- **🚨预警**：目标没转译就甩给执行；没给工具只给压力。
- **📌留痕**：启动会/目标同步会纪要 + 全员 todo。
- 🔗 配合 [[26.02 运行管理]] A2 需求传达8要素。

## B3. 对客预期管理（专业与主动权）

- **场景**：与客户确认目标、过程沟通。
- **核心逻辑**：预期管理本质是「让客户感到可控」而非「一味满足」。
- **✅标准动作（4步）**：① 目标拆解（客户目标→可执行可衡量的阶段目标，展示专业建立信任）② 信心输出（明确哪些有把握/有风险/需配合，不盲目承诺也不过度悲观）③ 持续教育（告知达成路径/所需支持/可能障碍，降低不合理预期）④ 拉回主动权（主动提建议方案而非被动接需求，建立顾问角色非执行角色）。
- **🚨预警**：客户目标无限放大、被动接单、不敢提风险。
- **📌留痕**：确认邮件/微信留痕（目标+操作路径+验收标准）。

# C. 目标怎么确认

## C1. 阶段目标调整 SOP

- **场景**：目标初定 / 触发调整时。
- **✅标准动作（7阶段→输出物）**：

| 阶段 | 关键动作 | 输出物 |
|---|---|---|
| 目标初判 | 拆解核心指标 | 目标初判文档 |
| 向上对齐 | 与+1/老板确认可行性+资源 | 目标确认纪要 |
| 目标拆解 | 拆阶段目标+执行动作+成本调整 | 目标拆解表 |
| 向下同步 | 启动会/目标同步会 | 纪要+全员todo |
| 对客确认 | 书面确认目标+路径+验收 | 确认邮件/微信 |
| 可视化落地 | 上墙/上文档/上群公告 | 看板/文档 |
| 动态调整 | 按反馈/数据变化调 | 目标调整通知 |

- **🚨调整触发条件**：客户明确变更 / 核心指标连续2周期未达 / 满意度预警 / 成本资源剧变 / 业务侧战略调整。
- **📌留痕**：每阶段对应输出物。

## C2. 自查 checklist（→ Phase 3 做成模板）

- **目标设定**：是否 具体/可衡量/可达成/相关/有时限；是否覆盖四侧；是否与内部目标冲突且有解；是否已量化为指标。
- **向上同步**：是否书面确认 / 说明资源需求和风险 / 获得资源承诺 / 有调整预案。
- **向下同步**：全员收到并拉齐 / 转译为可执行动作 / 明确谁负责谁配合谁兜底 / 提供工具 / 设检查节点 / 明确反馈机制红线。
- **对客确认**：书面确认 / 验收标准对齐 / 已管理预期（风险障碍提前告知）/ 有阶段汇报安排。
- **可视化**：进了群公告 / 写入SOP脚本 / 体现在汇报模板 / 上墙看板。

## C3. 🚫 七条红线（禁行）

| 红线 | 一句话 |
|---|---|
| 变更不留痕 | 任何调整必截图/邮件/微信确认，口头视为无 |
| 下达不复述 | 团队必须二次确认理解一致 |
| 调整不同步 | 目标变了立刻群公告@全员，别指望"自己会知道" |
| 风险不预判 | 目标阶段就做风险预判+处理办法，别等爆雷 |
| 执行不对齐 | 目标要具体到数字/案例/截止时间 |
| 承诺不留底 | 答应客户的写进文档，别靠脑子记 |
| 资源不评估 | 加目标前先算人力成本，别硬接 |

> 总诀：不只留口头、不默认已知、不隐瞒风险、不复述不确认。

## 🔗 skill 调用索引（Phase 3 节点触发用）
- 立项/接 brief → A1 目标统一 + 目标公式 + 四侧
- 向老板汇报 → B1 七维度（结论先行+数据+方案）
- 启动会派活 → B2 七要点（转译+边界+工具）
- 对客确认目标 → B3 四步 + C1 对客确认留痕
- 目标要变 → C1 调整SOP + C3 红线自查
- 任何阶段收尾 → C2 自查 checklist
