# 26.00 研究框架与路线图

> 「我们逐步学习」的方法论：怎么把 PM 工作标准研究透、并长成一个 loop skill。
> 总入口 → [[26 项目管理PM]]

---

## 一、研究框架（四层流水线）

研究不是「读完文档」，而是把素材逐层蒸馏成可执行的能力。每一层有明确的输入、动作、产出：

| 层 | 输入 | 动作 | 产出 | 落在哪 |
|---|---|---|---|---|
| **A 信源层** | 原始素材 | 收集、登记、标状态 | 信源清单 | [[26.09 信源与进度]] · `_信源/` |
| **B 知识层** | 信源 | 拆解 → 原子化 → 归入5模块 | 原子笔记 + 模板 | 26.01~26.05 · `_模板/` |
| **C 工具层** | 知识层 | 把 checklist/SOP/公式固化成可调用资产 | summer-pm-loop-skill skill | `~/.claude/skills/summer-pm-loop-skill` |
| **D 实践层** | 真实项目 | 跑 loop → 复盘 | 复盘案例 → 反哺信源层 | `_复盘案例/` |

**闭环**：D 产出的真实复盘会变成新的信源 A，再蒸馏进 B，迭代 C。越用越准。

---

## 二、知识原子化标准（B 层怎么记）

每条知识尽量收敛成一个「可被直接调用」的单元，统一结构：

```
### 知识点名称（动词开头更好：如「拆解brief」「下达需求」）
- 场景：什么时候用
- ❌ 常见误区：一句话
- ✅ 标准动作：步骤 / 公式 / checklist（这是 skill 真正要调用的部分）
- 🚨 预警信号：出现什么就要停 / 升级
- 📌 留痕物：产出什么文档
```

> 原则：**能变成 checklist 或公式的，就不要写成段落。** skill 调用的是「动作」，不是「道理」。

---

## 三、五大模块 = 知识的骨架

来源于两份文档的天然结构，已在 [[26 项目管理PM]] 列出。三句话记住它们的关系：

- **目标管理**定方向（做对的事）→ **运行管理**保过程（把事做对）→ **毛利经营**算账（事做得值不值）。
- **风险与救火**是横切的护栏，任何阶段偏了都靠它兜底。
- **复盘沉淀**是飞轮，把每次实战变成下一次的标准。

生命周期映射：**事前（启动）→ 事中（运行·七维盯盘）→ 事后（复盘）**，七维 = 内容/团队/客户/进度/数据/成本/卡点。

## 四、loop skill 设计（C 层的目标形态）

`summer-pm-loop-skill` 不是一次性问答，而是**绑定一个项目、周期性运行**的助手。设想的触发器：

| 节律 | 触发 | skill 干什么 | 调用的知识 |
|---|---|---|---|
| **每周** | 定时 loop | 七维健康盘点 + 毛利6数 → 出预警清单 | 26.02 七维 / 26.03 盘点 / 26.04 火苗 |
| **节点触发** | 接到 brief | brief必问清单 + 拆解5要素 + 一句话复述 | 26.02 brief |
| **节点触发** | 要派活 | 需求传达8要素 + 二次确认话术 | 26.02 传达 |
| **节点触发** | 提审前 | 内容7维校验 + 客户challenge自问 | 26.02 校验 |
| **节点触发** | 客户变更/不满 | 变更四件套 / 救火12步 checklist | 26.02 / 26.04 |
| **每月** | 月底 | 报数+报销提醒、毛利盘点模板 | 26.03 |
| **项目结束** | 收尾 | 心流复盘模板 → 写入 `_复盘案例/` → 提迭代SOP建议 | 26.05 |

数据流：skill **读** 知识层（动作库）+ 项目台账，**写** 复盘案例与预警记录。
loop 机制：用 `/loop`（自定步频）或 scheduled-tasks 定时跑「健康盘点」。

> ⚠️ skill 先不急着建。**知识层没夯实就做 skill = 空壳。** 见下方路线图，Phase 3 才动 skill。

## 五、路线图（逐步学习的节奏）

> 每个 Phase 是一次或几次对话的工作量。✅完成 / 🔄进行中 / ⬜未开始

- ✅ **Phase 0 · 搭骨架**（本次）
  建文件夹、定研究框架、出 MOC 索引、5 模块骨架、信源台账。

- 🔄 **Phase 1 · 吃透两份主文档 → 原子化**
  把《PM工作标准》《主R毛利SOP》逐节蒸馏成 26.01~26.05 的原子笔记（按上面的原子化标准）。一次深挖 1 个模块，避免贪多。
  *建议顺序：02 运行管理（最厚）→ 01 目标管理 → 03 毛利经营 → 04 风险救火 → 05 复盘。*

- ⬜ **Phase 2 · 补充信源**
  拉取飞书文档（lark-doc）+ 解析 `主R财务知识解析.pptx`，把财务/毛利部分加深，补进 26.03。

- ⬜ **Phase 3 · 固化模板 → 起 skill v0.1**
  把高频 checklist/公式/话术做成 `_模板/` 可复用件；搭 `summer-pm-loop-skill` skill 骨架，先实现 1~2 个节点触发（brief把控 + 内容7维校验）。

- ⬜ **Phase 4 · 接真实项目跑 loop**
  选一个在跑的项目当试验田，建项目台账，让 skill 做每周健康盘点；用实战检验知识缺口。

- ⬜ **Phase 5 · loop 自动化 + 飞轮**
  定时触发盘点与月度提醒；项目结束自动产出复盘并提 SOP 迭代建议，反哺知识层。
