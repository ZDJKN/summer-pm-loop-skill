# 26.02 运行管理 ⚙️

> 每天怎么干活才不乱。本质 = **过程标准化管理与控制**，拒绝「凭感觉」，一切看数据和事实。
> 上级 → [[26 项目管理PM]] ｜ 信源：《PM工作标准》01 运行管理（+ 飞书 PM工作标准类文档待并）
> 🟢 Phase 1 已原子化｜格式：场景 / ❌误区 / ✅标准动作 / 🚨预警 / 📌留痕

---

## 总纲

- **运行管理本质**：不是「盯人」，是建立 **早发现·早预警·早纠偏** 机制，防小偏差拖成大事故。
- **生命周期**：事前（启动·定目标防未然）→ 事中（运行·七维盯盘）→ 事后（[[26.05 复盘沉淀]]）。
- **七维盯盘**：内容 / 团队 / 客户 / 进度 / 数据 / 成本 / 卡点。
- **四铁律**：数据驱动 · 闭环管理 · 预警前置 · 书面留痕。

> 本模块分两大块：**A 内容管理（管事·保质量）** + **B 人员管理（管人·提效能）**。
> 成本相关的三隐形坑见 [[26.04 风险与救火]]；目标相关见 [[26.01 目标管理]]。

# A. 内容管理 —— 管事·保质量

## A1. 拆解 brief（把模糊变具体）

- **场景**：接到任何需求的第一步。做项目第一步不是排活/出创意，是回答「brief 在说什么 / 没说清什么」。
- **❌误区**：信息多≠清晰；急着安排工作；把客户的「表面说法」当「真实需求」。
- **✅标准动作**
  - **必问4维**：①需求理解（能一句话复述+客户确认）②边界（列「做/不做」清单）③标准（交付标准+验收标准/参考案例）④资源（对照历史同类项目评估够不够）
  - **解读4步**：先判断这是什么事 → 想客户真正要什么（"想做TVC"≠拍片，可能是新品造势/品牌升级）→ 标定限制条件（内容/品牌/预算/时间/人力/渠道边界）→ 判定成功标准（什么叫完成/做好/谁验收/按什么验收）
  - **拆解5要素**：识别问题 / 补全信息 / 校准认知 / 明确边界 / 建立标准
  - **要素具体化**：抽象词落到执行维度。"年轻人"→18-25岁女性·一二线·高频小红书；"高级感"→极简·冷色调·慢节奏·留白30%；"效果好"→播放≥10万·互动≥5%
  - **收尾必做（二次确认）**：一句话复述「这次核心是为达到__效果，面向__人群，__时间内输出__内容，传递__概念，以__验收，请确认」
  - **信息清晰度盘点 + 安全/预警水线**：把 brief 信息分三堆——**清楚的 / 不清楚要对齐的 / 要预警的**。对「频率没写明」的内容类型（场景种草、探厂、对谈等重制作最易漏），别拍一个数就算毛利，给客户**两条水线**：安全水线（≤N条/周团队可健康运转、毛利成立）/ 预警水线（>M条/周或每月固定X次→成本超支、毛利打穿，需加人/加预算/移出立项）。把三堆+两线写给客户用频率确认，毛利测算才真实。
  - **多需求单独拆**：多个账号/需求不要塞进一个文档，单独拆、聚焦主账号，其余只作背景信息输出。
  - **文档结构重心前置**：①一句话结论（做什么事+问题点+可执行空间）②信息盘点与预警（清楚/要对齐含水线/风险边界，按P0/P1/P2分级，每条带解决思路）③下一步动作（核心重点在哪+指令1234）④目标与交付边界 ⑤KPI与毛利测算 ⑥风险与确认QA ⑦内容策略与栏目（可薄）。意义=不让读者再拆一层，直接给拆解结果，所以最前面必须先定调。
- **🚨预警**：自己复述不出核心需求；说不清「不做什么」；标准停留在"好看/要爆/越快越好"；**对频率未明的内容直接拍一个数算毛利**。
- **📌留痕**：brief 拆解文档（重心前置结构）+ 客户确认记录。

> 简单需求是「翻译」（客户知道要什么），复杂需求是「共创」（真需求在反复沟通里长出来）。
> 思考路径：读懂 → 问清 → 拆解 → 量化 → 确认。
> 🔗 接 brief 后算接不接/报价 → [[26.03 毛利经营]]：人力按团队滚动算（非单条）、毛利测算别漏隐藏人力。

## A2. 传达需求（把信息变共识）

- **场景**：brief 拆完，要把需求下发给执行团队/外协。
- **❌误区**：你以为讲清楚了≠对方听懂；对方说"收到"≠理解一致；想到哪说到哪。
- **✅标准动作**
  - **传达8要素**：① 项目基础信息（名称/客户/对接人/阶段/状态）② 需求背景（brief原文/核心诉求/已确认方向）③ 交付标准（合格样子/客户偏好/参考案例/验收）④ 人员分工（谁主责/谁协作）⑤ 时间节点（交付/客户预期/留调整/发布）⑥ 风险预判（每个风险→影响什么结果→怎么处理→谁介入）⑦ 历史问题（客户雷区）⑧ 应急联络（什么情况找谁，10分钟无响应升级谁）
  - **动作链（缺一不可）**：结构化传达 → 书面留痕 → 二次确认（让接收方复述核心/最不能错的点/时间/风险）→ 全员同步共识 → 关键节点进度跟进
- **🚨预警**：只在私聊传达、群里没同步共识；接收方没复述就开工。
- **📌留痕**：需求单/派活消息/AI纪要（群消息、文档、邮件、录音均可）。

## A3. 内容质量不跑偏（把「我觉得过」变「标准过」）

- **场景**：内容提交客户**之前**的内部体检。
- **❌误区**：「我觉得差不多了」——"我觉得"不是标准，"差不多"不是交付质量；靠多改几版救。
- **✅标准动作**
  - **参考先行**：视觉/风格类需求必找参考图/片，严禁凭想象空做。
  - **内容7维校验**（不合格不提交）：① brief匹配（概念/策略/质感/结构/色感/字体）② 创意性（符合当前social环境）③ 品牌价值点（传递了某一价值点）④ 品牌传播策略（匹配当前阶段传播目标、有传播抓手）⑤ 品牌调性（不割裂人设）⑥ 风险可控（无舆情/版权/广告法隐患或有预案）⑦ 客户challenge自问（能达成目标吗/符合品牌吗/有硬伤吗/用户看懂了吗）
  - **通用客户challenge自答 q1-q6**：目标达成 / 品牌匹配 / 用户视角 / 竞品对标 / 风险排查 / 落地可行性——答案模糊就改。
- **🚨预警**：方向性返工、审美性争议、细节性错误、落地性问题。
- **📌留痕**：7维校验记录（提审 checklist）。

## A4. 提意见（把问题变方案）

- **场景**：review 内部稿 / 给客户反馈意见。
- **❌误区**：只说"感觉不对/再高级一点/我不太喜欢"——只指问题不给方向，对执行无帮助。
- **✅标准动作**
  - **公式**：哪里有问题 + 为什么是问题 + 建议怎么改 + 改完达到什么效果（现象/原因/建议/预期）
  - 例：❌"画面感觉不对" → ✅"第3秒背景太暗主体弱化（现象），色调古早显廉价不符高端定位（原因），提亮曝光+加背景高光参考demo（建议），效果是明快多巴胺、光影打在人脸（预期）"
  - **给客户提意见**（需求不一定对时）：分析利弊 → 专业建议 → 提示风险/预期 → 尊重决策（**必留痕**）。不硬顶也不盲从。
- **🚨预警**：反馈里出现"感觉不好/不够高级/再大气点/你再调调/还是差点意思"。
- **📌留痕**：意见记录（尤其给客户的专业建议+风险提示）。

## A5. 返工调整（把问题变沉淀）

- **场景**：内容被打回、反复修改。
- **❌误区**：惯性"再改一版看看/先顺着客户做"——返工>2次还硬改，只会人力浪费+情绪恶化+节奏失控+信任下降。
- **✅标准动作（返工>2次立即叫停 6步）**：① 叫停（通知全员暂停，主R收口）② review找因（不找背锅人：brief没读透？传达失真？标准模糊？质量不到位？客户变更？）③ 出新参考（按解法找参考/AI出demo）④ 拉齐（带demo/量化标准书面确认"这个可以吗"）⑤ 重启（确认后再投人力）⑥ 复盘（沉淀原因、优化SOP）
  - **四类返工对症**：理解偏差（复述+确认再传达）/ 标准模糊（补参考+量化+验收标准）/ 需求变更（标明是"变更"非"错误"，评估影响+书面确认+重排）/ 能力不足（换人/外部支持/重新协商标准）
- **🚨预警**：同一处改了2版+；客户说"差点意思"但说不清哪不对。
- **📌留痕**：review纪要 + 重新拉齐的确认 + 复盘条目。

## A6. 客户需求管理（把「要更多」变「要更对」）

- **场景**：客户中途加需求/提变更。
- **❌误区**：客户随口一说PM马上改；变更没评估影响；口头确认事后不认账；不合理也硬接。
- **✅标准动作（5步）**：① **暂停**（不秒答"好的"，回"我理解了，先评估工作量，xx时间内给具体回复"）② 评估**三影响**（时间：哪个节点delay？人力：谁做他在忙啥？效果/成本：影响原目标吗？）③ 给**选择题非判断题**（"可以做，方案A按您说的但xx delay 2天/方案B先做核心剩余下期，您看哪个"）④ **四件套留痕**（需求变更内容+影响说明+新排期+双方确认，形式不限）⑤ **同步三人**（团队怎么分工 / +1 若影响其他项目或成本 / 自己记下月底复盘）
  - 判断：三影响都小→直接接并同步；任一影响大→带方案谈条件。
- **🚨预警**：一周变更>2次（需求没问透）/ 总临时提马上要（客户缺计划性，需教育）/ 方向反复横跳（内部意见不统一，需明确唯一决策人）。
- **📌留痕**：变更四件套（微信/邮件/钉钉/录音）。

## A7. 合规审查（把风险变安全）

- **场景**：内容发布前。
- **✅标准动作（5维过检）**：① 广告法（禁用词：最/第一/顶级，夸大、绝对化）② 版权（素材/字体/音乐/BGM/插画/表情包/人像）③ 舆情（敏感议题/性别地域职业年龄刻板印象/隐私：手机号地址订单号聊天记录头像姓名）④ 平台规则（时长/格式/标签）⑤ 基础检查（错字/病句/数据错/链接错/logo错）
  - 舆情方式：发布前从「品牌/用户/舆论环境」三视角交叉判断；涉用户信息必脱敏；不确定先升级。
- **🚨预警**：一个错别字毁专业感、一个错链接漏流量、一个错logo触发品牌方不满。
- **📌留痕**：合规自查记录。

# B. 人员管理 —— 管人·提效能

> 目的：项目在动态变化中始终 **有人做·做得动·接得上·扛得住**，不因单人出问题就断电。

## B1. 协作流程（规范作业链）

- **场景**：多角色串联作业（客户/商务/PM/策略/文案/设计/剪辑/运营/投放/数据/法务/外协）。
- **❌误区**：信息在不同人手里断掉；前置没完成就开工；任务群里转没人收口；版本反馈散落；插单/跨部门就失控。
- **✅标准动作（5动作）**
  - ① **先清作业链**：接需求→拆解判断→传达→审核校验→对客提审→改稿→发布→回收数据→复盘；每环明确配合什么/到什么程度/产出物/截止/不配合怎么升级。
  - ② **明确每环「进入条件」+「交付物」+「谁确认」**：brief未拆清不进创意；参考未对齐不大规模出稿；初稿未7维校验不提客户；客户反馈未二次确认不分头改；定稿未明确不进发布；数据口径未统一不对外汇报。
  - ③ **关键节点有唯一收口人**：需求/brief拆解/内部方向/客户反馈/版本管理/上线版本/复盘结论。收口=确保信息统一、版本统一、结论形成、下一步明确。
  - ④ **固定协作动作**：需求到达先思考再拆解再分发（不直接转发聊天记录）；关键作业前先同步背景标准节点风险；群内>3轮无结论立刻拉会；会后必出纪要；客户反馈统一收口再同步；上线前最终版核对；上线后数据回收+异常复盘。
  - ⑤ **外协纳入同一流程**：需求传达/素材交付/标准确认/反馈收口/交付验收/风险升级都要约定，落「外协合作SOP」。
- **🚨失控场景→处理**：需求变了执行链没同步（立即暂停人力+统一同步+明确旧停新生效时间点）/ 上下游顺序错（回主作业链确认前置，重定串行并行，保关键路径）/ 多人回客户没收口（明确唯一对客收口，反馈先归类细节/方向/新增/风险再分发）/ 版本混乱用错稿（统一命名+版本号+终版标记，对客版与工作版分开）。
- **📌留痕**：项目SOP（标准链路）+ 各节点纪要。

## B2. 职责边界（RACI · 作业责任链）

- **场景**：分工定责、出问题找环节。
- **❌误区**：以为别人会做→没人负责；"共同负责"=无人负责；出事找不到责任人；多头沟通口径乱。
- **✅标准动作（3动作）**
  - ① **RACI 到人**：**R**esponsible 执行（可多人，**必有1个主R**）/ **A**ccountable 担责决策（一般小组长，每环节有唯一A）/ **C**onsulted 咨询（+1或跨部门专业意见）/ **I**nformed 知悉（上级/跨部门领导）。
  - ② **明确「边界外动作」+ 上报机制**：客户超范围新增/涉预算变化/涉人力重排/涉重大发布时间/涉品牌平台风险/连续不过审数据异常舆情 → 必须停手上报。**上报**：【给谁】直属→项目→部门（按层级）【时效】发现后30分钟内【结构】问题+影响范围+已尝试处理+需要什么支持。
  - ③ **建作业责任链（非责任点）**：每环节唯一A；出问题先归类属哪环节→找该环节A→A定处理路径→需要时升级→解决后复盘补SOP。
- **🚨预警**："我以为他在做"（拉会重确认RACI）/ 频繁甩锅（review补责任链）/ 对外口径不一（立即明确唯一对客收口）/ 客户跨层下任务（明确对接流程内部收口）/ 执行者事事请示（明确"能自己决定"范围）/ 决策者被琐事缠（下放执行层决策权）。
- **📌留痕**：项目SOP（责任链）+ 启动会RACI确认。

> 新认知：责任链固定到人，出问题先确认环节、再由环节责任人牵头解决——不是红卫军式批判追责，是让工作路径更顺。

## B3. 人力备案前置（消除单点依赖）

- **场景**：项目启动 / 进入高峰 / 角色调整 / 关键成员请假离职。
- **❌误区**：把备案当"形式化登记"；项目对特定人依赖过高（"只有他知道""得等他回来"）。
- **✅标准动作（4问 + 标准动作）**
  - **备案4问**：现在用了哪些人 / 各负责什么占多少精力 / 哪些是关键岗位谁可替补 / 有人缺勤项目怎么不断档。
  - **关键岗位判断**：此人离线1天会否直接影响 客户沟通/作业推进/数据回收/资料调用/决策判断——会，就是关键岗位，必须备案。常见：唯一对客窗口、唯一懂客户偏好的人、关键主创、数据运营、外协接口人。
  - **替补三层**：日常替补（请假调休）/ 紧急替补（突发病假失联）/ 能力替补（原能力不达标时谁接管）。
  - **资料备案**：素材库 / 源文件 / 执行文档（brief/方案/纪要/复盘）/ 对客记录（沟通截图确认）/ 账号权限——统一进小组知识库或项目文件夹，别散在私人聊天框。
  - **标准动作**：启动时即有初步备案 → 明确关键岗+替补 → 同步资料路径权限 → 关键节点前做一次人力风险复盘。
- **🚨预警**：某关键信息只1人知道；有人请假项目明显推进困难；大家都说"得等某某"；文件/账号依赖个人；换人后断层严重。→ 立刻补替补、开会拉齐、必要时向上申请资源。
- **📌留痕**：人力备案表 + 资料库路径清单 + 客户画像文档（偏好/雷区/历史争议/对接节奏）。

## B4. 团队负荷与情绪（守住可持续输出）

- **场景**：长跑项目、高压节点、多人并行。
- **❌误区**："年轻人扛一扛""大家都忙谁不累""先做完再说""没提离职就没事"——过载/情绪积压会反噬成执行失误、判断失真、服务感变差。
- **✅标准动作**
  - **负荷5观察维**：任务量（是否超载）/ 难度（是否匹配能力）/ 密度（阶段冲刺还是长期满载）/ 切换成本（频繁跨项目跨任务）/ 风险程度（总是同一批人扛最难最急最易背锅）。好分配 = 量配能力（能者多劳但多酬多成长）+ 难度配经验 + 节奏有缓冲 + 高风险有人兜底。
  - **情绪4动作**：① 及时识别（不等崩溃：异常沉默/表达欲降/抱怨增/机械交付/抗拒接活）② 区分「人的问题」还是「机制问题」（先修机制：目标/排期/分工/人力/修改机制/反馈是否合理）③ 给反馈不只给压力（明确哪里好/贡献在哪/及时认可，让成长可视化）④ 高压节点主动减震（统一收口客户反馈、挡无效打扰、先拆优先级再下任务、能扛的外部压力别原样转嫁）。
  - **PM角色**：做压力的**过滤器**，不是放大器。能扛的自己扛，必须传递的说明原因背景。
- **🚨预警**：连续加班无缓冲 / 返工率升 / 低级错误增多 / 沟通耐心降 / "怕看消息" / 频繁说"随便都行你定" / 输出越来越保守 / "做什么都没用"的无力感。
- **📌留痕**：负荷/情绪一对一沟通记录（必要时）+ 机制调整项。
