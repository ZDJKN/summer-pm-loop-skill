# 26.03 毛利经营 💰

> 从「执行思维」转向「经营者思维」。**你的价值需要用数字证明，保护自己不被替代。**
> 一句话流程：客户提需求 → 我们算利润 → 问清楚 → 出方案 → 报价 → 系统立项 → 干活 → 每月报数+报销。

## 核心公式（统一口径 · skill 直接调用）

| 概念 | 公式 | 例 |
|---|---|---|
| 不含税收入 | 含税 ÷ 1.06（税率6%） | 10万 ÷ 1.06 ≈ 9.43万 |
| 人力成本 | 工资 + 公司部分五险一金 | 月薪1万+公司交4k = 1.4万 |
| 统一分摊人力 | 总后台成本 ÷ 项目数 | 后台10万/10项目 = 每个摊1万 |
| 项目毛利 | 不含税收入 − (项目人力+外包+直接花费) | 9.43万 − 2.75万 = 6.67万 |
| 毛利率 | 毛利 ÷ 不含税收入 × 100% | ≈ 70.8% |
| 人效比 | 月毛利 ÷ 团队人数 | 5人月毛利20万 = 每人4万 |
| 考核毛利 | 毛利 × (1 − 10%)（扣风险备用金） | 6.67万 × 0.9 = 6万 |
| 人力成本利润率 | 总毛利 ÷ 总人力成本 | 20万 ÷ 14万 = 1.43 |

> **接单红线：毛利率 < 30% 不接；运行中 < 20% 亮红灯。**

## C1. 项目承接全流程（别瞎答应）

- **场景**：从甲方需求到立项。
- **✅标准动作（7步→注意点）**：

| 步 | 做什么 | 别踩坑 |
|---|---|---|
| 1 甲方需求 | 接需求 | 先判断能不能落地，别瞎答应 |
| 2 测算毛利率 | 算能赚多少 | Excel模板填，<30%不接 |
| 3 QA沟通 | 问清模糊点 | **书面确认**，避免后期扯皮 |
| 4 输出策划案 | 写执行方案 | 基于确认后的共识，别自己发挥 |
| 5 输出报价单 | 列收费明细 | 不超预算，覆盖成本+利润 |
| 6 CRM立项 | 系统建项目拿订单号 | **必须有订单号才能干活** |
| 7 执行监控 | 干活盯KPI | 每月底报收入/成本给财务 |

- **🚨预警**：⚠️ **CRM 订单上的执行时间，是花钱的唯一依据**——确认前不动任何花钱动作（投放/采购/外包），客户催提前就要求改合同或书面确认，否则超出时间甲方不认、团队自掏腰包。
- **📌留痕**：QA书面确认 / 策划案 / 报价单 / CRM订单号。

## C2. 每月盘点 6 数（自己不算·没人替你算）

- **场景**：每周更新、月底出结果。
- **❌误区**：等月底收到亏损通知才后悔；用合同额算（应用实际完成/到账）。
- **✅标准动作（6个数）**：① 当月收入(不含税：投放看消耗金额，内容按KPI完成进度)② 人力成本(每人在本项目工时，每周记一次月底汇总)③ 外包成本(剪辑/设计/开发，留发票付款记录)④ 其他成本(差旅/物料/软件/投放，按项目归属分摊别混)⑤ 毛利=①−②−③−④(正数才赚钱)⑥ 毛利率=⑤÷①(<20%红灯)。
  - 节奏：每周花10分钟更新一张表，月底自动出结果。
- **🚨预警**：账面亏损（先查是不是人力硬摊）/ 利润虚高（先查是不是漏了预提）。
- **📌留痕**：收入预提成本明细表（模板：某外贸平台客户执行收入、预提成本明细.xlsx）。

## C3. 每月必做两件事

- **① 报数给财务**（月底前）：本月实际收入 + 成本 + 人力分摊 + CRM确认当月权责。
- **② 报销+付款**：员工垫付月底前报；外协费用月底前1天走完申请（走外协SOP）。
- **📌留痕**：人力填报表 / 成本费用月度预提表 / 收入预提明细。

## 踩坑速查（结算/立项/对账三板块）

### D1. 结算：表面KPI≠真目标
- **坑**：只盯甲方表面KPI（曝光），没搞清真实目标（线索/签单）→ 执行完不满意、钱收不回、断档。
- **✅解法**：开工前问清最终想要什么（线索？签单？注册？），执行中持续对齐复盘。
- **记**：表面KPI不是真目标，签单才是验收单。*案例：某外贸平台客户从只教拍剪→以线索为目标，月6万→370万。*

### D2. 立项：花钱看订单 + 财务不认"差不多"
- **坑1**：商务没和甲方确认执行起止时间，团队提前花钱 → 超出时间甲方不认，自掏腰包。**解**：CRM订单确认前不动花钱动作。
- **坑2**：预算表命名不规范/小数点不保留2位/格式不对 → 财务反复打回卡2-3天。**解**：提前拿财务模板，逐项核对命名/小数位/必填字段。
- **记**：财务系统不认"差不多"，少一位都退回。

### D3. 每月对账三大坑
| 坑 | 后果 | 解法 | 一句话 |
|---|---|---|---|
| 人力分摊（多项目按工时硬摊） | 预算紧的项目账面亏损 | 按预算承受能力摊，不足不硬摊，提交前查异常亏损 | 人力按预算摊，不是按人头分 |
| 预提成本（漏已发生未付款） | 利润虚高、下月对不上、财务追责 | 月底让成员上报"已发生未报销"，活干了就预提 | 没付款≠没花钱 |
| CRM月权责（投放拉错周期/算错） | 账目混乱财务核不上 | 建标准模板，平台拉+内部拉双核对，差异>5%先排查 | 投放数据是算出来的，模板+双核对缺一不可 |

### D4. 测算：人力别按单条算，毛利别漏隐藏人力
- **坑1（单条思维）**：把日更轻量内容按「单条制作」估人力 → 严重低估。真实人力是一个团队在**滚动推进**：今天发 + 囤明后天稿 + 切片分发抖音/小红书 + 平台差异化 + 图文更新。**"轻"只是制作方式轻，不代表人力压力轻。** 判断口径永远是「团队滚动时怎么配合」，不是「单条怎么做」。
- **坑2（频率黑洞）**：内容类型的频率没写明（场景种草=夜宵/露营/看球/轰趴配酒这类「场景+人员+制片」重活尤其）→ 单条成本和它吃掉多少人力的时间线全是未知，这块根本没进成本表。算出来的毛利是**乐观上限**。
- **坑3（拍脑袋编人力）**：自己假设团队人数和薪资去算毛利 → 数从根上就是假的。
- **✅解法**：① **人力用真实团队数据，拿历史同类项目的人力分配做基线，优先直接问人/问团队，其次抓项目台账，绝不编**；② 人力按团队滚动口径估，把分发/囤稿/差异化/图文都算进同一体系；③ 对频率未明的内容给**安全水线 / 预警水线**（见 [[26.02 运行管理]] A1），毛利按「预算档位 × 水线场景」出区间，不给单一确定值；④ 报价前把"哪些清楚、哪些不清楚、水线在哪"讲给客户，用频率回填后再定毛利。
- **记**：没给频率的内容，毛利测算就是不真实的。先要水线，再算钱。*案例：某酒类品牌客户 brief 未写场景种草频率 + 日更按单条估，初版毛利测算被判定不可信。*
