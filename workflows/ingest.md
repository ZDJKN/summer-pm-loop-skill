# 知识摄取管道（自主学习 · 抖音/PM文档 → 蒸馏 → okf 知识包）

> 让 skill **自己学**：抓抖音职场内容、PM 文档、外部 SOP，过一道**蒸馏闸门**，把「可执行的动作」并进 `knowledge/`，把「鸡汤/道理」丢掉。
> 触发：「学习一下这些资料 / 抓点 PM 内容学一学 / 把这个文档蒸馏进知识库」，或定时（cron）批量跑。
> 🔑 铁律：**不是抓来就进知识库**——闸门挡住稀释，知识库才不变水。

---

## 决策树

```
[1] 来源在哪？
     抖音/视频    → opencli（抓收藏夹/话题）→ 转写文本（whisper / lark-minutes）
     文章/网页    → opencli / WebFetch
     本地文档     → 直接读（用户给的 md/pdf）
        ↓
[2] 暂存：写进 knowledge/_信源/ingest/（带来源链接 + 抓取日期），先不进模块
        ↓
[3] 🚪 蒸馏闸门（一条一条过）——这条是「可执行动作」吗？
     ✅ 是 = 能变成 checklist / 公式 / 标准动作 / 话术  → 留，转下一步
     ❌ 否 = 只是道理 / 情绪价值 / 流量金句            → 丢（或只存信源区备查）
        ↓
[4] 归类 + 打标签双链：进对应模块的「候选区」
     目标→26.01 / 运行→26.02 / 毛利→26.03 / 风险→26.04 / 复盘→26.05 / 对客→26.06
     按原子格式写：场景 / ❌误区 / ✅标准动作 / 🚨预警 / 📌留痕，标 #来源
        ↓
[5] 人工 promote（半自动，防污染）：你看一眼候选 → 确认 → 并入正式模块
     不确认就停在候选区，绝不自动塞进真理来源
```

---

## 蒸馏闸门 · 一句话判据

> **这条信息，能让下次行动更对吗？** 能 → 进；只是「听着有道理」→ 丢。
- ✅ 例：「客户不回 → 给选择题 + 报自己默认动作」→ 可执行，进 26.06。
- ❌ 例：「沟通要用心，将心比心」→ 道理，丢。

## 为什么要「人工 promote」这一步

知识库是 skill 的**真理来源**，一旦污染，所有判断都歪。所以：
- 抓取 / 蒸馏可以自动跑（甚至定时），
- 但**并入正式模块前留一道人确认**——先小批量（10 条）验证蒸馏质量，靠谱了再放量。
- 这就是「**先稳判断，再上自动**」在知识层的体现。

## 自动化程度（诚实）
- 抓取 + 暂存 + 初筛 → **可自动 / 可定时**（cron 跑 opencli + 闸门初筛进候选区）。
- 并入真理来源 → **保留人确认**（防稀释）。
- 这套和 [`loop.md`](loop.md) 是两条线：loop 学「**项目动态**」，ingest 学「**通用方法**」，都回流到知识包。
