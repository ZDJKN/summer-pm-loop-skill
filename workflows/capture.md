# 抓取工作流（loop 阶段① · onboard 一个项目）

> 把一个项目的「人/事/流程/背景」抓进 `projects/<项目>/`，喂给阶段② 盘点。
> 触发：用户说「onboard 这个项目 / 帮我把这个项目建档 / 盘一下 X 项目」且档案还不存在时。
> 前置：先跑过 [preflight](preflight.md)，知道有没有可用连接器。

---

## 决策树

```
开始（要 onboard / 盘点项目 X）
│
├─[1] projects/X/ 存在吗？
│      存在 → 跳到「增量更新」（只抓最近变化，回写 diff）
│      不存在 → 新建 projects/X/（复制 _TEMPLATE）→ ↓
│
├─[2] X 的信息在哪？（看 preflight 探到的连接器 + 问用户）
│      飞书 → lark-cli：im 抓群成员+消息、base 读表、doc 读文档
│      钉钉 → dws
│      Slack/Notion/Jira → 对应 MCP
│      没有连接器 → 让用户粘贴群消息/表格/截图
│
├─[3] 抓什么 → 对齐动态知识包字段
│      · 人  → 群成员/参与者 → 02 人.md（拆 RACI + 客户画像：偏好/雷区/对接节奏）
│      · 事  → 最近 N 条消息 / 表格 → 03 事.md（进度 + 待确认）+ 05 卡点.md
│      · 流程→ 节点/排期/作业链 → 04 流程.md
│      · 背景→ 合同/目标/KPI/brief → 01 背景.md
│
├─[4] 写进 projects/X/01~06（来源全部标注：哪条消息/哪张表，可回溯）
│
└─[5] 触发阶段② 盘点：读 knowledge 给 X 体检
       → 七维健康灯 + 卡点 + 下一步 + 预警 → 回写 00 项目总览.md
```

---

## 🔀 抓取路径（开局二选一 · 由 preflight 帮你定）

先看：项目在哪个平台、要不要微信、要不要无人值守，再选路径。**默认先用 🅰 轻路径**。

### 🅰 轻路径（默认 · 推荐先用 · 对同事最友好）
- **飞书** → `lark-cli`（**扫码登录**即可，不用起服务）
- **钉钉** → `dws`
- **调度**：preflight 探到 cron/launchd → 可定时；没有 → 半自动（你喊「跑今天盘点」）
- 适合：主要在飞书/钉钉、能接受「用时喊一句」、想让**不太懂的同事也好上手**（登录就用，零部署）。

### 🅱 cn 路径（按需升级 · 部署较重）
- 装 [cn-messaging-context](https://github.com/Iii3pl/cn-messaging-context)：后台持续抓 + 入库 + 摘要 + 定时。
- **只在以下才上**：① 要抓**微信**群；② 要**无人值守全自动**（不开 agent 也自抓）；③ 要**团队共享一个信息库** / 跨平台身份。
- 代价：起服务 + 配各平台 webhook/凭证（对不太懂的同事门槛高）。

> 选哪条由 preflight「就绪报告」给推荐；**不确定就先 🅰，撞到墙（要微信/要全自动）再上 🅱**。
> 无论 🅰 还是 🅱，**抓回来的信息一律按下面的方式落地**——路径只是「怎么拿」，存法是统一的。

---

## 📥 统一落地：标签 + 双链（不随路径变）

**信息「怎么拿」可以二选一，但「怎么存」永远一样**——抓回来的信息一律按这套存进 `projects/<项目>/`：

- **双链** `[[ ]]`：项目信息回指知识库与彼此。例：05 卡点里写「卡在 [[26.04 风险与救火]] 等待坑」、02 人里 [[某客户画像]]。
- **标签** `#`：按 `标签体系.md` 的四维打——`#节点/对客` `#类型/话术` `#阶段/事中` 等，让「所有救火话术 / 某项目所有卡点」能一键聚合。
- 这样无论 lark-cli 拉的、cn 摘要的、还是你手动粘的，进了档案就是**同一张信息 map**，可检索、可回溯、可被盘点调用。

## 抓取要点（别踩坑）

- **只抓、不编**：抓不到的字段留空标「待确认」，绝不替用户脑补客户信息（呼应核心理念：书面留痕、数据驱动）。
- **标来源**：每条写进档案的信息标注出处（消息 ID / 表名 / 用户提供），方便复盘回溯。
- **隐私**：真实档案只写本地 `projects/`，`.gitignore` 已挡，**永不进公开仓库**。
- **降级**：没连接器就请用户粘贴，照样能 onboard，只是阶段①从自动变手动。
- **增量**：已建档的项目，每次只抓「自上次盘点后的变化」，在 00 总览里写「本次变了什么（diff）」——这就是 loop 阶段③ 滚动。

---

## 与盘点的衔接（阶段①→②）

抓完立刻进 [盘点]：拿 `knowledge/26.02`（七维）+ `knowledge/26.04`（火苗信号）逐维体检，输出到 `00 项目总览.md` 的健康灯与预警。这一步**不需要任何外部工具**，纯读知识库 + 项目档案。
