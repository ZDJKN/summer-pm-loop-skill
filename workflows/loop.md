# Loop 工作流（v2 · 滚动盘点 / 多重视角 / 辅助决策）

> **v2 的本质不是「定时发日报」**，而是把项目信息**持续变成判断和决策支持**：提炼结论、**多重视角**、辅助决策。日报只是副产品。
> **信息从哪来 = 二选一**（见 [`capture.md`](capture.md)：🅰 轻路径 lark-cli/dws / 🅱 cn）。loop **不绑死**任何一种——拿数据的方式可换，判断和落地方式不变。

---

## 双层架构

```
信息源（二选一，见 capture）
  🅰 轻路径：lark-cli(飞书扫码)/dws(钉钉)/手动粘贴   ← 默认
  🅱 cn-messaging-context：后台持续抓+摘要+定时       ← 要微信/全自动/团队共享时
        │  （怎么拿可换）
        ↓ 结构化上下文
判断层 / 大脑   summer-pm-loop-skill（读 knowledge）
  提炼结论 · 多重视角 · 健康盘点 · 辅助决策
        ↓ 写（标签 + 双链，统一）
projects/<X>/（动态知识包） + 预警 + 决策建议
```

---

## loop 每 tick

对每个在册项目 `projects/<X>/`：
1. **取增量**：按 capture 选定的路径拿自上次以来的变化（🅰 lark-cli/手动现拉 · 🅱 cn 摘要）→ 更新 01–06（**标签+双链**落地）。
2. **多重视角盘点**（见下）→ 重写 `00 总览`。
3. **出 diff**：`scripts/pm_loop.py --diff` → 写「本次变化」+ 预警。
4. **红灯/升级 → 显式抛人**（不只写文件）。

## 🎯 多重视角（v2 核心能力 · 不只一个健康灯）

同一份项目上下文，过 5 个知识库镜头，**各出一句判断 + 该谁动**——这才是「辅助决策」：

| 视角 | 镜头（knowledge） | 它回答 |
|---|---|---|
| **客户** | 26.06 / 26.04 | 客户在想什么？有火苗吗？响应/情绪如何？ |
| **团队** | 26.02 B | 谁过载/卡住？有单点依赖吗？ |
| **向上** | 26.01 B1 | 投产比/风险边界，要不要升级老板？ |
| **成本** | 26.03 | 毛利健康吗？有没有隐藏人力？ |
| **交付** | 26.02 七维 | 卡点在哪？进度对得上吗？ |

输出 = 「五视角各一句结论 + 下一步该谁动」，决策者一眼看清「最该担心哪个、谁去动」。
> 多重视角**不依赖 cn**——只要有信息（哪怕手动粘的）就能跑。

---

## 自适应：谁来跑 + 何时触发（harness 无关）

判断层不绑死 agent（Claude Code / Codex / cron 都行）。由 [`preflight.md`](preflight.md) 探测后选触发器：
- 🅱 **cn 的定时日摘要**到了 → 直接触发（最省，仅 cn 路径有）。
- 系统 **cron / launchd** → 定时调 `scripts/pm_loop.py --due` → 逐个盘（🅰 也能用）。
- **Claude Code** → `/loop` / scheduled-tasks。
- 都没有 → **半自动**：你说「跑今天的盘点」。

## 降级：永远不崩

信息源任一缺失，最低退到「你粘信息进来」，多重视角照常跑。判断层**永远独立可用**；cn 只是把「连续 + 微信 + 跨平台 + 团队共享」加上去。

---

## 🚫 红线
- **自动 ≠ 无人看**：红灯/升级类预警必须显式抛给人。
- **抓不到就降级手动，绝不编数据填健康灯**（假绿灯比没灯更危险）。
- **先稳判断，再全自动**：上定时无人值守前，先半自动陪跑几轮，确认多视角判断靠谱。
