# 安装后引导（v1.3 · 自检 → 装齐 → 上 cn / 兜底轻路径）

> 装好 / 部署这个 skill 后跑一次。目标：**检查环境和工具够不够支撑 loop + 信息处理**，缺的提示装，让安装者决定，**多提醒几次「装齐了效果和自学习才完整」**。
> 两种跑法：① 安装者在终端跑 `bash scripts/setup.sh`（逐项问、能自动装）；② 或 agent 按下面决策树口头带着走。

---

## 决策树

```
[1] 核心（必须）：knowledge/ 5+ 模块 + scripts 在？
       缺 → 包不完整，停，重新获取
[2] 体验 & 自学习：Obsidian 装了吗？
       没装 → 提醒「它是看 标签+双链 信息 map 的窗口，soul 自学习成果靠它可视化」
            → 问装不装；装 → brew install --cask obsidian
       ⚠️ 重要性提醒①：没它能用，但自学习沉淀看不见，效果打折
[3] 抓取：node/npm（cn 必需）、lark-cli（飞书）、dws（钉钉）、cron（定时）逐个探
       缺 node → 问装；缺 lark-cli → 提示去装（轻路径要用）
[4] cn（推荐 · loop 的信息核心）：
       讲清为什么值得装——它让 skill 从「你喊才跑」变「自己持续抓+喂判断」，
       自学习(soul + 知识摄取) 和 多视角盘点 才真跑得起来。
       ⚠️ 重要性提醒②：这是 v1.3 的灵魂，强烈建议上
       → 问装不装 → 装：git clone + npm install + build
            成功 → 提示还差「配各平台凭证/webhook + 注册 MCP」(只有安装者能做)
            失败 → 不报错退出，转 [5] 兜底
[5] 结论：
       cn 就绪 → 列「配凭证 + 注册 MCP」两步，配完即全自动
       cn 没上/失败 → 🅰 轻路径兜底：lark-cli 扫码/dws + (cron定时 or 喊一句) + 标签双链落地
                     并告诉他「等真要微信/无人值守，再回来上 cn」
       ⚠️ 重要性提醒③：装得越全(Obsidian+cn)，多视角盘点和自学习越完整，别只用半套
```

## 引导原则

- **让安装者决定，但把「为什么重要」说够**：每个可选项都讲清「装了 skill 多出什么效果 / 自学习多跑得起来」，**全程提醒 ≥3 次**（Obsidian 一次、cn 一次、结尾一次）——因为不装齐，skill 只是半套。
- **优先上 cn**（全自动 + 微信 + 自学习闭环），但**cn 装不上/跑不通绝不卡死**：立刻给「我的轻路径」兜底方案 + 具体执行步骤（见 [`capture.md`](capture.md) 🅰 / `setup.sh` 结论区）。
- **先问再装**，不静默动用户机器；但缺核心依赖(node 等)要明确指出否则 cn 上不了。
- 装完引导他做一次「首次使用」环境自检 + 「盘一下 _DEMO」看效果，形成正反馈。

## 与其它工作流的关系
- 本流程是**一次性安装引导**；日常用 [`preflight.md`](preflight.md)（每次开局自检）。
- cn 上不上，决定 [`loop.md`](loop.md) 是「无人值守全自动」还是「半自动轻路径」。
